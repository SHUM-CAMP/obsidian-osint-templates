# SHUM Obsidian OSINT Templates

Practical Obsidian templates and note structures for OSINT documentation, investigation workflows, and knowledge management.

This repository is a SHUM-adapted fork of [WebBreacher/obsidian-osint-templates](https://github.com/WebBreacher/obsidian-osint-templates). We use it as a foundation for building templates, note structures, and workflows tailored to SHUM’s investigative and analytical work.

## About SHUM

[SHUM](https://shum-ng.org/) is a public organization founded by experts from the international OSINT community. The organization uses OSINT methods to investigate war crimes related to Russia’s war against Ukraine, support law enforcement and judicial actors, counter disinformation, and contribute to human rights work.

More about SHUM:
- Website: https://shum-ng.org/
- LinkedIn: https://www.linkedin.com/company/ngoshumcamp/

## Purpose of this repository

This repository serves as a working base for:
- documenting investigations in Obsidian
- structuring notes, entities, and case files
- supporting SOP-driven workflows
- extending the original templates with SHUM-specific materials
- developing new templates for real operational and analytical tasks

The goal is not to preserve the original vault unchanged, but to use it as a solid starting point for further development.

## What is included

The repository currently includes:
- reusable templates for common OSINT note types
- example structures for linked notes and case documentation
- Standard Operating Procedure materials
- SHUM-specific additions and future customizations

## Suggested use

This repository is intended for teams and practitioners who want to use Obsidian as a local, link-based environment for:
- OSINT documentation
- case and incident tracking
- evidence organization
- research workflows
- analytical note-taking
- internal knowledge management

## Repository structure

Typical sections in this fork include:
- `-- templates` — reusable note templates
- `Standard Operating Procedures` — SOP-related materials
- `examplecase` — example notes and structures
- `SHUM` — organization-specific additions

## Development approach

SHUM uses this fork as a foundation for further work. New templates and workflows may be added for:
- incidents and events
- persons, organizations, and accounts
- domains, infrastructure, and technical artifacts
- sources and evidence records
- geospatial and verification tasks
- analytical products and reporting workflows

General improvements that are useful beyond SHUM may be contributed upstream where appropriate.
