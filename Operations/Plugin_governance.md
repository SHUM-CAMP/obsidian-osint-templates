# Plugin Governance

## Allowlist policy
- Keep plugin list short and role-driven.
- Add plugins only when they support a defined workflow outcome.

## Selection criteria
- Active maintenance
- Export portability
- Low lock-in risk

## Degrade gracefully notes
- If `dataview` is unavailable, use static index notes.
- If `quickadd` or `templater` is unavailable, use manual template insertion.
- If `metadata-menu` is unavailable, edit frontmatter directly.
